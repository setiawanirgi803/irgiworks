## 5. Database Schema & RBAC Requirements

### Database (SQLite):
- Create a `projects` table: `id`, `title`, `category`, `description_detail`, `image_url`, `date_created`, `status`.
- Create a `users` table: `id`, `username`, `password_hash`, `role_id`.
- Create a `roles` table: `id`, `role_name` (e.g., 'SuperAdmin', 'Editor').

### RBAC Logic:
- **SuperAdmin:** Full access to view, create, edit, delete projects, and manage other user accounts.
- **Editor:** Can only view, create, and edit existing portfolio data (cannot delete projects or manage users).
- **Public:** Can only read/view public endpoints.

## 6. Performance & Security Best Practices
- **Images:** Implement auto-conversion or restriction to modern web formats (WebP/AVIF) and implement responsive image sizing (`srcset`).
- **Caching:** Implement lightweight server-side caching or SQLite query optimization for portfolio fetching.
- **Security:** Hash admin passwords using safe algorithms (like bcrypt/argon2), implement strict session management, and sanitize inputs to prevent SQL Injection and XSS.
